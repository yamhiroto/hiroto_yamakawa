Problem Definition¶

The dataset I use is collected from The U.S. Department of Transportations (DOT) Bureau of Transportation Statistics, which contains more than 5 million records of the flights information during 2015, including the carrier of the airline, the original airport, destination airport, the distance of the flight, the time spent, the departure delay time, and arrival delay time, etc.

Dataset can be downloaded on Kaggle website.

https://www.kaggle.com/usdot/flight-delays

I will use this dataset to train a model to predict if a flight will be delayed or not.

create the machine learning pipeline¶

Step 1 - Convert all the categorical data in the dataset into numerical data (since the underlying machine is more familiar with numbers than text) : For the flights dataset, we have three categorical columns: AIRLINE, ORIGINAL_AIRPORT and DESTINATION_AIRPORT. Therefore,we construct three StringIndexer objects.

Step 2 - The OneHotEncoderEstimator object will convert all the categorical columns into their corresponding numerical columns with sparse vector representations(a sparse vector representation is a more efficient way to represent a vector if the majority of the vector column is zero values)

Step 3 - the VectorAssembler will allow us to merge all the sparse vectors and other numerical values in the dataset into a single sparse vector, by concatenating them one by one and call this new vector "features". We won't include "Arrival_delay" to prevent Data leakage

Step 4 - We will compare three different classifiers : DecisionTree, Random Forests and Logistic Regression.

(I tried tuning the models with the ParamGridBuilder and a 5-fold CrossValidator each time, but for that size of dataset my laptop wasn't powerful enough)

Step 5 - we will evaluate the performance of each model/ As we have a binary classification problem, this can be done by comparing the Area Under ROC Curve, precision, recall, and F1-scores.

#compute TN_lr,TP_lr,FN_lr,FP_lr
predictions_lr.groupBy('label','prediction').count().show()

# Calculate the elements of the confusion matrix
TN_lr = predictions_lr.filter('prediction= 0 AND label = prediction').count()
TP_lr = predictions_lr.filter('prediction= 1 AND label = prediction').count()
FN_lr = predictions_lr.filter('prediction= 0 AND label <> prediction').count()
FP_lr = predictions_lr.filter('prediction= 1 AND label <> prediction').count()

# calculate accuracy, precision, recall, and F1-score
accuracy = (TN_lr + TP_lr) / (TN_lr + TP_lr + FN_lr + FP_lr)
precision = TP_lr / (TP_lr + FP_lr)
recall = TP_lr / (TP_lr + FN_lr)
F =  2 * (precision*recall) / (precision + recall)

print(f'n precision: {precision}')
print(f'n recall: {recall}')
print(f'n accuracy: {accuracy}')
print(f'n F1 score: {F}')

Conclusion Random Forest did a terrible job.

Even for Logistic Regression which is the best of the three model, accuracy is not very satisfying. We could improve our results by :

  • increasing the quantity of Data : we have currently used the data for 2015 only, whereas the original website has data from 1994.

  • tuning the hyperparameter with a Gridsearch and Cross-Validation process.

  • adding additional features. Right now we only used info such as departure time, source and destination airports to do our prediction on flight delays. we could add information about the weather, location of the airport (GPS coordinate), special events calendar.
